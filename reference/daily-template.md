# 日报模板（v1.2）

## 标题格式

`YYYY-MM-DD AI 日报`

---

## 正文结构

### 1. 今日重点（S / A 级）

按重要性排序，不强控数量。选取今天最值得读的 2-4 条（论文 / 行业动态均可），每条需完整展开。

**论文类格式：**

```markdown
**[标题](链接)** `[领域标签]` ⭐ 顶会标记（如有）
- 🏛️ 机构｜作者1, 作者2 et al.
- ❓ 工程问题：现有方案在工业落地/实际系统中的哪个具体痛点
- 🔧 核心方法：用什么技术手段解决，一句话，点到本质
- 💡 对你的价值：对推荐/大模型/搜索/用户反馈工程有什么直接参考价值
```

**行业动态类格式：**

```markdown
**[标题](链接)**
- 来源：来源名（日期）
- 一句话摘要：发生了什么，为什么重要
- 信号：对算法工程 / 科技产业意味着什么
```

---

### 2. arxiv 论文精选

覆盖领域：大模型 / 推荐系统 / 生成式推荐 / 用户反馈 / 搜索与检索。
cs.IR 全收 + cs.AI 关键词过滤后打分排序，合并取 Top 10。

**⚠️ 每篇论文必须包含：**
1. **作者 + 机构**（格式：`作者 et al.｜机构`，机构尽量写全，区分学术机构与工业界）
2. **中文详细摘要**（3-5 句话，覆盖：背景痛点 → 核心方法 → 关键创新点 → 实验结果）
3. **领域标签**

每条格式：

```markdown
- **[标题](https://arxiv.org/abs/xxxx.xxxxx)** `[大模型]` / `[推荐]` / `[搜索]` / `[用户反馈]`
  - 🏛️ 机构（学校/公司，必填，优先级高于作者名）｜作者1, 作者2 et al.
  - ❓ 工程问题：现有方案在工业落地/实际系统中的哪个具体痛点（不是学术背景）
  - 🔧 核心方法：用什么技术手段解决，一句话，点到本质
  - 💡 对你的价值：这个工作对推荐/大模型/搜索/用户反馈工程有什么直接参考价值
```

⚠️ **摘要是工程意义，不是翻译 abstract。** 站在算法工程师视角说清楚：解决了什么工程问题、核心方法是什么、对你的实际工作有什么参考价值。禁止写纯数字提升但无工程结论的摘要。

### 6. 我的观察

用 1-3 段话总结。**不得省略，每次必须生成。** 

写作规范：
- 不是每条新闻的复述，是跨条目的横向判断
- 点明今天最值得盯的主线（通常 1-2 条）
- 说清楚为什么这些信号对算法工程 / 科技行业有判断价值
- 如有跨条目的共鸣或反差，主动点出来
- 指出哪些值得明后天继续跟踪

示例（风格参考）：

> 今天两条核心主线：
>
> **生成式推荐**：ComeIR 和 FedMM 都是 S 级。前者把 SID 量化的两个根本问题拆得很清楚，dual-level memory 是关键创新；后者腾讯出品、SIGIR 2026，跨市场 CTR + 联邦的组合有直接工业落地价值，值得找代码读。
>
> **Frontier-Eng Bench 指向的方向**——深度迭代推理 > 宽度探索——和今天 arxiv 上 RPSFT 的思路有某种共鸣：都在说"不要靠多而靠精"。

### 7. 元信息

```markdown
- 生成日期：YYYY-MM-DD
- 信息源数量：N 个
- arxiv 原始条目：N 条 → 过滤后 N 条 → 入选 N 条
- 行业/媒体原始条目：N 条 → 入选 N 条
- 总入选：N 条
```

## 生成原则

- 强排序，不强控条数
- **论文必须带作者+机构，摘要用中文写，不少于 3 句话**
- **⚠️ 机构信息由 `fetch_arxiv.py` 的 `fetch_affiliations()` 自动补全，AI 直接使用脚本输出的 `affiliations` 字段，不需要自己逐篇抓 `/html/` 页面**
- **⚠️ 若脚本输出的 `affiliations` 为空但 `authors` 字段中包含机构文本，AI 应从 `authors` 中提取机构，不得写「机构待确认」**
- **科技快讯必须覆盖所有已抓取源（36Kr、极客公园、The Verge、爱范儿等），不得遗漏**
- 链接统一使用 Markdown 可点击格式
- 优先保证覆盖和层次，而不是过度压缩
